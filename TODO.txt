🔒 SECURITY ISSUES & RECOMMENDATIONS:

CRITICAL - SECRETS EXPOSURE:
-----------------------------
1. ⚠️ APP_SECRET is EMPTY in .env file
   - File: .env (line 28)
   - FIX: Generate a strong secret using `php bin/console secrets:generate-keys` or `openssl rand -hex 32`
   - This is used for CSRF tokens, remember-me cookies, and other security features

2. ⚠️ Mercure JWT secret uses default value "!ChangeThisMercureHubJWTSecretKey!"
   - Files: .env (line 22), compose.yaml (lines 8, 9, 15)
   - FIX: Generate a unique secret for production

3. ⚠️ Database password hardcoded in backup script
   - File: scripts/backup.ps1 (line 18)
   - FIX: Use environment variables instead: $env:MYSQL_PASSWORD

HIGH PRIORITY:
--------------
4. 🔐 No login rate limiting / brute force protection
   - The login form has no rate limiting to prevent brute force attacks
   - FIX: Install symfony/rate-limiter and configure login throttling in security.yaml
   - Example: Add login_throttling: { max_attempts: 5, interval: '15 minutes' } to form_login config

5. 🔐 No security headers configured
   - Missing Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, X-XSS-Protection
   - FIX: Add security headers in Caddyfile or via Symfony NelmioSecurityBundle
   - Only Permissions-Policy for browsing-topics is set in Caddyfile

6. 🔐 isActive check not enforced at login
   - CustomUserProvider loads user but doesn't check if isActive is false
   - A deactivated user could still log in with valid credentials
   - FIX: Add isActive check in CustomUserProvider.loadUserByIdentifier() or use a UserChecker
